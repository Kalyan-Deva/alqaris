CVE-2016-8375 // WI-FI COMPACTFLASH CARD SWAP
CompactFlash
Card Replacement
Attack
Affects: BD Alaris 8015 v9.7 and 8000 PC unit
01
Gain Physical Access
Attacker reaches the Alaris 8015 at bedside, in a hallway, storage room, or during device maintenance. No authentication required — the attack is entirely physical. Typical hospital deployments leave devices unattended between patient use.
02
Swap the CompactFlash Card
The external CompactFlash slot houses the Wi-Fi adapter card. The attacker pops it out and inserts a malicious logging CF card — indistinguishable externally. Takes under 10 seconds. No tools needed.
03
Device Writes Credentials to Card
On reconnect, the Alaris 8015 v9.7 writes the hospital Wi-Fi SSID and PSK to the CF card in plaintext — no encryption, no obfuscation. The malicious card captures and stores this data silently.
04
Retrieve Card → Network Access
Attacker retrieves the card on a second visit (or uses a wireless exfil variant). They now hold the hospital's MedNet Wi-Fi password and can join the clinical network from any nearby location — car park, lobby, or adjacent building.
CVE-2016-9355 // INTERNAL FLASH MEMORY EXTRACTION
Internal Flash
Chip Removal
Attack
Affects: BD Alaris 8015 firmware ≤ v9.5
01
Access and Disassemble Device
Attacker accesses the pump and disassembles the casing using a standard Phillips screwdriver. The Alaris 8015 has no tamper-evident hardware in baseline configuration — BD's only mitigation is to add tamper-proof stickers after the fact.
02
Remove Internal Flash Module
The internal flash chip is a removable module — not soldered to the PCB. The attacker lifts it out of its socket in seconds. This chip stores Wi-Fi credentials, network configuration, and sensitive technical data in plaintext.
03
Extract Data Offsite — Zero Detection Risk
Unlike CVE-8375, the attacker takes the chip away from the hospital entirely. In their own lab, they mount it in a CH341A programmer, run binwalk to analyze firmware, and use HxD hex editor to locate and read plaintext credentials.
04
Full Hospital Wi-Fi Access + Pivot
Armed with extracted SSID and PSK, attacker authenticates to MedNet from any coverage location. From inside the VLAN, they can reach other infusion pumps, EHR servers, DICOM imaging systems, and pharmacy drug libraries.
PHASE 3 // POST-EXPLOITATION — HOSPITAL NETWORK BREACH

STEP 01 // AUTHENTICATE
Join the MedNet VLAN
Using the extracted PSK, the attacker's device authenticates to the hospital's clinical Wi-Fi network. The access point has no way to distinguish them from a legitimate medical device. They are now inside the firewall perimeter.
VLAN: 192.168.10.0/24
STEP 02 // RECONNAISSANCE
Scan for Clinical Targets
A rapid nmap scan reveals the full attack surface: other infusion pumps, EHR servers, nurse station terminals, DICOM imaging systems, and pharmacy servers — all on the same flat medical device network.
14 pumps + 8 servers found
STEP 03 // PIVOT + IMPACT
Lateral Movement to Critical Systems
Attacker pivots across the network. Potential targets include: modifying pump drug libraries to alter medication dosing, accessing patient EHR records (HIPAA violation), disrupting DICOM imaging delivery, or establishing persistent backdoor access.
