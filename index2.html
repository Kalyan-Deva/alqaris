Vulnerability Deep Dive

Two unpatched attack vectors enabling unauthorized network access through physical device compromise.

CVE-2016-8375

Wi-Fi Card Replacement Attack

Affected: Version 9.7 and 8000 PC unit

  • Attacker swaps the external CompactFlash Wi-Fi card with a malicious logging card — no special tools needed
  • Device writes unencrypted Wi-Fi credentials (SSID + PSK) to flash storage accessible via the CF interface
  • Extracted credentials allow the attacker to authenticate to hospital Wi-Fi (MedNet) and pivot to clinical systems
  • Attack requires physical access; performing it in a clinical setting increases detection probability
CVE-2016-9355

Internal Flash Memory Extraction

Affected: Version 9.5 and prior

  • Attacker physically disassembles the Alaris 8015 to access the internal removable flash memory module
  • Older firmware (≤9.5) stores Wi-Fi credentials and sensitive data on removable flash — no encryption
  • Removable format enables extraction away from the facility, eliminating detection risk
  • Standard flash readers (CH341A) read memory directly; credentials stored in plaintext

Our Methodology

Three-phase structured approach to vulnerability reproduction and impact analysis.

Phase 1

Hardware Acquisition & Recon

  • Source BD Alaris 8015 unit from secondary market
  • Identify CF slot location and accessibility
  • Document external ports and interfaces
  • Map physical security controls
Phase 2

Vulnerability Reproduction

  • Swap CF card with proxy logging device
  • Disassemble unit to access internal flash
  • Use binwalk + HxD for memory analysis
  • Extract and document plaintext credentials
Phase 3

Impact & Defense Analysis

  • Simulate VLAN pivot using extracted creds
  • Assess clinical network access scope
  • Propose operational and technical defenses
  • Compile findings into final report
